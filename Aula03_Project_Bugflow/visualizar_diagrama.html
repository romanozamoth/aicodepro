Explicação do Fluxo de Processamento

  1. Entrada do Bug: Um novo bug ou incidente é registrado no sistema.
  2. Banco de Dados Neon: Os dados do bug são armazenados no banco PostgreSQL da Neon.
  3. Indexação Pinecone: Paralelamente, o conteúdo textual do bug é convertido em embeddings e indexado no Pinecone para pesquisas semânticas e identificação de bugs similares.
  4. Processamento por Agentes:
    • AG1 (Classificador de Componente): Analisa o bug e determina qual componente do sistema está afetado (Frontend, Backend, Database, etc.).
    • AG2 (Classificador de Severidade): Avalia o impacto e determina a severidade do bug (Crítico, Grave, Menor).
    • AG3 (Analista Técnico de Bugs): Realiza análise técnica detalhada, identifica causas raiz e sugere soluções.
    • AG4 (Gerenciador de Resolução): Coordena a resolução do bug e atribui a desenvolvedores apropriados.
    • AG5 (Documentador de Bugs): Cria documentação completa em formato Markdown e HTML interativo com diagrama Mermaid.
  5. Documentação Final: Documentação completa do bug, incluindo análise, solução, testes e lições aprendidas, é gerada em formato Markdown e HTML interativo com diagrama Mermaid.

O diagrama também mostra as interações com:

  • OpenAI API: Fornece os modelos de IA para todos os agentes.
  • Vector Store (Pinecone): Armazena embeddings para consulta semântica.
  • DB Relacional (Neon): Armazena todos os dados estruturados dos bugs e suas resoluções.
